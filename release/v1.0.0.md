# ContextTrace v1.0.0

ContextTrace 1.0 is the first stable release of the local-first evidence-chain
forensics SDK and CLI for RAG and AI agents.

## Highlights

- Trace failures from retrieval and selected context through claims, citations,
  evidence spans, verdicts, root causes, repair guidance, and regression tests.
- Diagnose RAG and agent traces locally with lexical, semantic, local-ML, NLI,
  or explicitly configured judge modes.
- Capture existing endpoints, compare runs, audit retrieval, generate static
  HTML reports, and replay saved failures in CI suites.
- Integrate with LangChain, LlamaIndex, FastAPI, LangGraph, and OpenTelemetry.
- Keep SQLite state, traces, reports, caches, and judge data local by default.
- Separate grounding, truth, source authority, freshness, and citation status
  so evidence support is not presented as independent correctness.

## Benchmark Evidence

The versioned benchmark archive contains the checksummed RAGTruth, ARES, CRAG,
and ContextTrace-Diag-150 evidence used for the current readiness report.

The broad SOTA gate remains fail-closed at `8/10`: primary RAGTruth independent
review and Diag-150 independent sign-off are still open. This release therefore
claims a stable, benchmarked product, not broad state-of-the-art performance.

## Install

```bash
pip install contexttrace==1.0.0
contexttrace init
contexttrace verify-demo unsupported_claim --report
```

## Upgrade

```bash
python -m pip install --upgrade contexttrace==1.0.0
contexttrace --version
contexttrace doctor
```

The stable release preserves the existing local-first CLI and SDK behavior from
the 0.9 series while formalizing the supported public surface.
