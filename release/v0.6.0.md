# ContextTrace v0.6.0

ContextTrace v0.6.0 adds a complete local evidence QA workflow for RAG outputs.

This release moves ContextTrace beyond separate debugging commands. Developers can now capture a RAG response, inspect trace quality, verify claim-level evidence, audit retrieval and corpus failures, and produce one local QA report with risk level and recommended next actions.

## Highlights

- **Evidence QA workflow** with `contexttrace qa trace.json --corpus docs/ --report`.
- **Trace inspection** with `contexttrace inspect trace.json` to check extracted claims, duplicate context IDs, citation references, warnings, and suggested next commands before verification.
- **Endpoint capture** with `contexttrace capture endpoint` for turning one live RAG API response into portable verification JSON.
- **Saved response capture** with `contexttrace capture response response.json` for debugging responses collected from logs or failing runs.
- **Risk scoring** in QA output, including risk level, primary issue, risk signals, and CI gates.
- **Stronger retrieval audit diagnostics** with failure stages, evidence status, diagnostic signals, failure paths, developer summaries, and prioritized recommended actions.
- **Local QA HTML report** covering inspection warnings, claim verification, optional corpus audit, risk signals, and next actions.
- **Validation pack docs** for recording public-source RAG app runs and saved-response debugging workflows.
- **Public app validation harnesses** for public-source RAG app response shapes and a local LangChain/Ollama/FAISS end-to-end run.

## Core Commands

```bash
contexttrace capture endpoint \
  --endpoint http://localhost:8000/query \
  --query "What is the refund policy?" \
  --out traces/refund_trace.json \
  --verify \
  --report

contexttrace capture response response.json \
  --query "What is the refund policy?" \
  --out traces/refund_trace.json \
  --verify \
  --report

contexttrace inspect traces/refund_trace.json

contexttrace qa traces/refund_trace.json \
  --corpus docs/ \
  --report
```

## QA Failure Gates

```bash
contexttrace qa trace.json --corpus docs/ --fail-on high_risk
contexttrace qa trace.json --corpus docs/ --fail-on medium_risk
contexttrace qa trace.json --corpus docs/ --fail-on audit_failure
contexttrace qa trace.json --fail-on inspect_warning
```

Supported QA gates:

- `high_risk`
- `medium_risk`
- `any_risk`
- `unsupported`
- `should_abstain`
- `audit_failure`
- `inspect_warning`

## What This Release Helps Diagnose

- unsupported claims
- missing or weak retrieved evidence
- citation mismatches
- retrieval misses
- chunking issues
- reranking issues
- corpus gaps
- answer overreach
- stale or conflicting source evidence
- answers that should have abstained

## Limitations

- Default verification remains local and heuristic.
- Claim extraction is still rule-based.
- Contradiction detection is conservative.
- Semantic matching is local normalization, not embeddings or an LLM judge.
- ContextTrace is built for debugging and regression checks, not replacing human review in high-stakes domains.
