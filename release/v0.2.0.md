# ContextTrace v0.2.0

ContextTrace v0.2.0 adds local claim-level evidence verification for portable RAG traces.

The release focuses on one core question: is each generated claim actually supported by retrieved evidence?

## Highlights

- **Portable verifier** with `contexttrace verify trace.json`.
- **Bundled proof demos** with `contexttrace verify-demo`.
- **Local HTML verification reports** with claim verdicts, citation status, evidence snippets, highlighted matched terms, and suggested fixes.
- **Atomic claim decomposition** for simple compound claims.
- **Support verdicts**: `supported`, `partially_supported`, `unsupported`, `contradicted`, and `unverifiable`.
- **Citation mismatch detection** for missing sources and cited sources that do not support the claim.
- **Abstention judgment** for answers that should not have made factual claims from the retrieved context.
- **CI failure gates** through `--fail-on`.
- **Verification modes**: deterministic `lexical` mode by default and local paraphrase-aware `semantic` mode.
- **Bundled benchmark** through `contexttrace verify-benchmark`, including exact-match rate and per-label precision, recall, and F1.

## Quickstart

```bash
pip install contexttrace
contexttrace verify-demo unsupported_claim --report
contexttrace verify-demo citation_mismatch --fail-on citation_mismatch
contexttrace verify-benchmark --mode semantic
```

## Portable JSON Input

```json
{
  "query": "How long does refund processing take?",
  "answer": "Refunds are processed within 5 business days.",
  "contexts": [
    {
      "id": "policy_2026",
      "text": "Customers may request refunds within 30 days of purchase."
    }
  ]
}
```

## CLI Commands

```bash
contexttrace verify trace.json
contexttrace verify trace.json --json
contexttrace verify trace.json --report --out reports/example.html
contexttrace verify trace.json --mode semantic
contexttrace verify trace.json --fail-on unsupported --fail-on citation_mismatch
```

Bundled demos:

```bash
contexttrace verify-demo unsupported_claim --report
contexttrace verify-demo partial_support --report
contexttrace verify-demo citation_mismatch --report
contexttrace verify-demo should_abstain --report
contexttrace verify-demo supported_answer --report
```

Benchmark:

```bash
contexttrace verify-benchmark --mode lexical
contexttrace verify-benchmark --mode semantic --json
```

## Verdicts

- `supported`: the retrieved evidence supports the claim.
- `partially_supported`: the evidence supports part of the claim, but not every detail.
- `unsupported`: no retrieved context has enough evidence for the claim.
- `contradicted`: conservative local checks found explicit negation or conflicting values.
- `unverifiable`: evidence overlap is too weak or ambiguous.

## CI Gates

```bash
contexttrace verify trace.json --fail-on unsupported
contexttrace verify trace.json --fail-on citation_mismatch
contexttrace verify trace.json --fail-on any_failure
```

Supported gates:

- `unsupported`
- `partial_support`
- `citation_mismatch`
- `should_abstain`
- `contradicted`
- `unverifiable`
- `no_citation`
- `any_failure`

## Limitations

- The default verifier is deterministic and lexical.
- `--mode semantic` uses local normalization, not embeddings or an LLM judge.
- Contradiction detection is intentionally conservative.
- Claim extraction is rule-based.
- ContextTrace is a debugging tool, not a replacement for human review in high-stakes domains.
