# ContextTrace v0.1.0

ContextTrace v0.1.0 is the first public release of the local-first SDK and CLI for evaluating existing RAG and AI agent systems.

It is built for developers who want to debug retrieval misses, unsupported claims, citation mismatches, context failures, and basic agent/tool failures without uploading private traces or documents to a hosted dashboard.

## Highlights

- **SDK tracing** for query, retrieval, selected context, answers, citations, token usage, latency, metadata, and agent events.
- **CLI evaluation workflow** with `contexttrace init`, `contexttrace demo`, `contexttrace eval`, `contexttrace benchmark`, `contexttrace report`, and `contexttrace doctor`.
- **BYO RAG endpoint evaluation** for testing an existing RAG API without installing the SDK into that app.
- **Local SQLite trace store** under `.contexttrace/contexttrace.db` by default.
- **Citation verification** with claim-to-source support verdicts.
- **RAG failure taxonomy** covering retrieval misses, citation mismatches, unsupported answers, contradictions, conflicting sources, abstention failures, and more.
- **Local HTML reports** for trace inspection and shareable debugging output.
- **Demo datasets** including refund policy, employee handbook, and AI paper QA examples.
- **GitHub Action regression checks** for failing CI on RAG quality regressions.
- **Integrations** for custom RAG pipelines, LangChain, LlamaIndex, FastAPI endpoint tracing, LangGraph beta tracing, and OpenTelemetry export.

## Quickstart

```bash
pip install contexttrace
contexttrace init
contexttrace demo --dataset refund_policy
contexttrace report --last --open
```

## Evaluate Your Own RAG API

```bash
contexttrace eval \
  --dataset evals/questions.json \
  --endpoint http://localhost:8000/query \
  --method POST \
  --input-key question \
  --answer-path $.answer \
  --contexts-path $.contexts \
  --citations-path $.citations \
  --fail-on "failure_rate>0.25"
```

## Notes

- Local mode is the default.
- No hosted dashboard is required.
- Network calls only happen when you configure an LLM judge provider or evaluate a RAG endpoint you provide.
- This release is intended as a developer-preview foundation for local RAG reliability workflows.
